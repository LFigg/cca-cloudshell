# Microsoft 365 Collector

The M365 collector (`m365_collect.py`) gathers resource inventory from Microsoft 365 including SharePoint, OneDrive, Exchange, Teams, and optionally Entra ID (Azure AD).

## Authentication Options

### Option 1: Azure CLI / Managed Identity (Recommended)

The simplest and most secure approach uses `DefaultAzureCredential`, which automatically authenticates using:
- **Azure Cloud Shell**: Uses your logged-in identity automatically
- **Local development**: Uses Azure CLI credentials (`az login`)
- **Azure VMs**: Uses Managed Identity

```bash
# Login with Azure CLI
az login

# Run collector
python3 m365_collect.py --use-default-credential

# Include Entra ID users and groups
python3 m365_collect.py --use-default-credential --include-entra
```

### Option 2: App Registration with Client Secret

For automated/service scenarios, use an Azure AD App Registration:

```bash
# Set credentials in terminal session (NOT in files)
export MS365_TENANT_ID="your-tenant-id"
export MS365_CLIENT_ID="your-client-id"
export MS365_CLIENT_SECRET="your-client-secret"

# Run collector
python3 m365_collect.py
```

> **Security Note**: Never save credentials to files or commit them to version control. Set environment variables directly in your terminal session.

## Command Line Options

| Option | Description |
|--------|-------------|
| `--use-default-credential` | Use Azure CLI / Managed Identity (recommended) |
| `--tenant-id ID` | Azure AD tenant ID (or use env var) |
| `--client-id ID` | App registration client ID (or use env var) |
| `--include-entra` | Include Entra ID users and groups |
| `--skip-sharepoint` | Skip SharePoint sites |
| `--skip-onedrive` | Skip OneDrive accounts |
| `--skip-exchange` | Skip Exchange mailboxes |
| `--skip-teams` | Skip Microsoft Teams |
| `-o, --output PATH` | Output directory |
| `--log-level LEVEL` | Logging level (default: INFO) |

## App Registration Setup

For Option 2 (client secret), you need an Azure AD App Registration with Microsoft Graph API permissions.

### Step 1: Create App Registration

1. Go to **Azure Portal** → **Azure Active Directory** → **App registrations**
2. Click **New registration**
3. Configure:
   - Name: `CCA-M365-Collector`
   - Supported account types: **Accounts in this organizational directory only**
   - Redirect URI: Leave blank
4. Click **Register**

### Step 2: Note the IDs

From the app registration **Overview** page, copy:
- **Application (client) ID** → `MS365_CLIENT_ID`
- **Directory (tenant) ID** → `MS365_TENANT_ID`

### Step 3: Add API Permissions

1. Go to **API permissions** → **Add a permission**
2. Select **Microsoft Graph** → **Application permissions**
3. Add these permissions:

| Permission | Purpose |
|------------|---------|
| `Sites.Read.All` | Read SharePoint sites |
| `Files.Read.All` | Read OneDrive files/storage |
| `User.Read.All` | Read user profiles & mailboxes |
| `Mail.Read` | Read mailbox metadata |
| `Team.ReadBasic.All` | Read Teams information |
| `Group.Read.All` | Read group membership |
| `Reports.Read.All` | **Recommended** - Enables fast bulk collection for OneDrive/SharePoint/Exchange via usage reports. Without this, collection falls back to slow per-user API calls (can take hours for large tenants) |
| `Organization.Read.All` | Read tenant licensing info (recommended) |
| `Directory.Read.All` | Read Entra ID (optional, for --include-entra; also works for licensing) |

4. Click **Grant admin consent for [Your Organization]**
   - Requires Global Administrator or Privileged Role Administrator

### Step 4: Create Client Secret

1. Go to **Certificates & secrets** → **New client secret**
2. Description: `CCA Collector`
3. Expiration: Choose based on your security policy
4. Click **Add**
5. **Copy the secret value immediately** (shown only once) → `MS365_CLIENT_SECRET`

### Step 5: Configure Environment

```bash
export MS365_TENANT_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
export MS365_CLIENT_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
export MS365_CLIENT_SECRET="your-secret-value"
```

Or pass via command line:
```bash
python3 m365_collect.py \
    --tenant-id xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx \
    --client-id xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx \
    --client-secret your-secret-value
```

## Collected Data

### Resources

| Resource Type | Service Family | Data Collected |
|---------------|----------------|----------------|
| `m365:sharepoint:site` | SharePoint | Site count, storage used (GB) |
| `m365:onedrive:account` | OneDrive | Account count, storage used (GB) |
| `m365:exchange:mailbox` | Exchange | Mailbox count, storage used (GB), item count, mailbox type |
| `m365:teams:team` | Teams | Team count (storage counted in SharePoint) |
| `m365:entra:user` | EntraID | User count (optional, for --include-entra) |
| `m365:entra:group` | EntraID | Group count (optional, for --include-entra) |

### Exchange Mailbox Types

The collector distinguishes between different mailbox types sourced from the Exchange usage report:

| Mailbox Type | Description |
|--------------|-------------|
| **User** | Standard user mailboxes |
| **Shared** | Shared departmental mailboxes |
| **Room** | Room mailboxes for meeting room scheduling |
| **Equipment** | Equipment mailboxes (projectors, vehicles, etc.) |
| **Group** | Microsoft 365 Group mailboxes |
| **Discovery** | eDiscovery mailboxes |

Additional mailbox metadata collected:
- **Has Archive** - Whether archive mailbox is enabled
- **Storage Quotas** - Warning, send, and send/receive quota limits
- **Quota Usage %** - Current usage as percentage of quota
- **Deleted Item Count/Size** - Recoverable deleted items
- **Last Activity Date** - Most recent mailbox activity
- **Created Date** - When the mailbox was created
- **Is Deleted** - Whether the mailbox is soft-deleted (in retention period)

### SharePoint Site Types

The collector classifies SharePoint sites by type based on the root web template:

| Site Type | Root Template | Description |
|-----------|---------------|-------------|
| **Personal** | SPSPERS | OneDrive personal sites |
| **Team Site** | GROUP, TEAMCHANNEL | SharePoint team sites |
| **Communication Site** | SITEPAGEPUBLISHING | Publishing/communication sites |
| **Group Site** | GROUP#0 | Microsoft 365 Group sites |
| **Other** | Various | Other site types (e.g., classic sites) |

Site metadata collected:
- **Is Deleted** - Whether the site is soft-deleted
- **Last Activity Date** - Most recent site activity
- **File Count** - Total number of files
- **Active File Count** - Files active in the last 30 days

### Teams Activity Metrics

The collector retrieves Teams user activity data:

| Metric | Description |
|--------|-------------|
| **Team Chat Messages** | Count of messages sent in Teams channels |
| **Private Chat Messages** | Count of messages sent in private/group chats |
| **Calls Count** | Number of calls made |
| **Meetings Count** | Number of meetings attended |
| **Active Users** | Count of users with Teams activity |

### Change Rate & Growth Metrics

The collector retrieves 180 days of historical usage data to calculate:

| Metric | Description |
|--------|-------------|
| **Daily Change (GB)** | Average daily data change volume |
| **Daily Change (%)** | Daily change as percentage of total storage |
| **Annual Growth (%)** | Projected annual growth rate based on historical trend |

These metrics are collected for SharePoint, OneDrive, and Exchange services.

## Troubleshooting

### "Insufficient privileges" Error

- Ensure admin consent is granted for all permissions
- Go to App registration → API permissions → verify green checkmarks

### "Invalid client secret" Error

- Client secrets expire - check the expiration date
- Create a new secret if expired

### Partial Data Collection

Test individual services:
```bash
# Test SharePoint only
python3 m365_collect.py --skip-onedrive --skip-exchange --skip-teams

# Test OneDrive only
python3 m365_collect.py --skip-sharepoint --skip-exchange --skip-teams
```

### Rate Limiting

Microsoft Graph has throttling limits. The collector handles basic retry logic, but very large tenants may experience delays.

| Feature | Required Permissions |
|---------|---------------------|
| SharePoint Sites | `Sites.Read.All` |
| OneDrive | `Files.Read.All`, `User.Read.All` |
| Exchange Mailboxes | `User.Read.All`, `Mail.Read` |
| Microsoft Teams | `Team.ReadBasic.All`, `Group.Read.All` |
| Change Rate & Growth | `Reports.Read.All` |
| Entra ID (optional) | `Directory.Read.All`, `User.Read.All`, `Group.Read.All` |
