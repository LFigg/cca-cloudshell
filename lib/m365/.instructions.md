# M365 Collection Module

## Overview
Microsoft 365 data collection via Microsoft Graph API. Collects inventory from Exchange, SharePoint, OneDrive, Teams, and Entra ID.

## Public API
```python
from lib.m365 import collect_m365, get_graph_client

# Full collection
result = collect_m365(config)

# Individual services
from lib.m365.exchange import collect_mailboxes
from lib.m365.sharepoint import collect_sites
from lib.m365.onedrive import collect_drives
from lib.m365.teams import collect_teams
from lib.m365.licensing import get_user_license_assignments
```

## Module Structure
| File | Purpose |
|------|---------|
| `auth.py` | Graph client initialization, credential handling |
| `exchange.py` | Mailbox collection, usage reports |
| `sharepoint.py` | Sites, libraries, storage |
| `onedrive.py` | OneDrive accounts |
| `teams.py` | Teams, channels, activity |
| `licensing.py` | SKUs, user license assignments |
| `reports.py` | Usage report parsing utilities |
| `helpers.py` | Graph API pagination, async helpers |

## Key Dependencies
- `msgraph-sdk>=1.0.0` - Microsoft Graph SDK
- `azure-identity>=1.12.0` - Azure authentication
- `httpx` - HTTP client for direct API calls

## Authentication Modes
1. **DefaultAzureCredential** - Azure CLI, Managed Identity, etc.
2. **ClientSecretCredential** - App registration with secret
3. **InteractiveBrowserCredential** - User login (for testing)

## Testing
```bash
pytest tests/test_m365/ -v
```

## Common Issues
- **"Event loop is closed"** → Use `run_sync()` helper from `helpers.py`
- **403 Forbidden** → Check Graph API permissions (User.Read.All, etc.)
- **Empty usage reports** → Tenant may have reports disabled/concealed

## Data Flow
```
Graph API → Raw Response → CloudResource → JSON Output
                ↓
         Usage Reports → Size/Activity Enrichment
```

## Output Format
All collectors return `List[CloudResource]` from `lib.models`.
